---

### 🔧 Step 1: Authenticate GitHub CLI
Make sure you’re logged in with your GitHub account:

```powershell
gh auth login
```

- Choose **GitHub.com**
- Select **HTTPS**
- Authenticate with your browser or a token

---

### 📦 Step 2: Create a New Repository
Run this command in PowerShell:

```powershell
gh repo create mj-ahmad --public --source=. --remote=origin --push
```

### 🧾 Breakdown:
- `mj-ahmad` → Name of your new repo (replace with your desired name).
- `--public` → Makes the repo public (use `--private` if you want it private).
- `--source=.` → Uses the current folder as the repo source.
- `--remote=origin` → Sets the remote name to `origin`.
- `--push` → Immediately pushes your local code to GitHub.

---

### 📂 Step 3: Initialize Locally (if not already)
If your folder isn’t yet a Git repo, initialize it first:

```powershell
git init
git add .
git commit -m "Initial commit"
```

Then run the `gh repo create` command above.

---

### ✅ Step 4: Verify
Check that the remote is set correctly:

```powershell
git remote -v
```

And confirm your repo exists online by visiting:

```
https://github.com/YOUR_USERNAME/mj-ahmad
```

### 🗂 Organizing Multiple Repos in One Location
Right now your folder `C:\Users\Print Gallery-4\OneDrive\Desktop\MJAhmad` contains several subfolders (`Draft`, `Note`, `Timeline`, `mjahmad`, etc.). Each of these can be its **own Git repository**, but you need to initialize them separately.

- **One repo per folder**:  
  Each project folder (`Draft`, `Note`, `Timeline`, etc.) should have its own `.git` directory inside it. That way, Git knows it’s a separate repo.

- **Your current setup**:  
  You have a `.git` folder at the **root** (`MJAhmad`). That means Git is treating the entire `MJAhmad` directory as **one big repo**, not separate ones.

### 🔄 Moving Between Repos
To work with different repos in the same location:

1. **Navigate into the repo folder**:
   ```powershell
   cd Draft
   ```
   or
   ```powershell
   cd Timeline
   ```

2. **Initialize or check repo**:
   ```powershell
   git init        # only if not already a repo
   git status      # check if Git recognizes it
   ```

3. **Add remote if needed**:
   ```powershell
   git remote add origin https://github.com/MJ-Ahmad/Draft.git
   ```

4. **Stage, commit, and push**:
   ```powershell
   git add .
   git commit -m "Initial commit"
   git push -u origin main
   ```

### 📌 Example Workflow
Suppose you want to push changes from `Note` and then from `Timeline`:

```powershell
# Work with Note repo
cd Note
git add .
git commit -m "Update notes"
git push -u origin main

# Switch to Timeline repo
cd ../Timeline
git add .
git commit -m "Update timeline"
git push -u origin main
```

### ✅ Key Takeaway
- Each subfolder must be its own repo if you want to push separately.  
- You move between them using `cd` into the folder.  
- Then run the usual `git add`, `git commit`, and `git push`.
